###### 4)BCNF --BOYCE CODE NORMALIZATION FACTOR.

###### BCNF rule says:

###### -The left side (determinant) must be a candidate key.

###### -REAL TIME PROBLEM SOLVE.



###### EX:-(NOT FOLLOWING BCNF)

###### STU\_ID  SUBJECT  TEACHER

###### &nbsp; 1      MATH     SHARMA

###### &nbsp; 2     SCIENCE   RIYAA

###### HERE,

###### Since Subject alone determines Teacher, the table violates BCNF. Subject is NOT the candidate key.Non-key column determining another column.So the table breaks BCNF rule.

###### 

###### NOW FOLLOWING BCNF:-

###### STU\_ID  SUBJECT

###### &nbsp; 1      MATH

###### &nbsp; 2    SCIENCE

###### 

###### SUBJECT   TEACHER
